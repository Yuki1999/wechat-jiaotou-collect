# 党政办信息跟踪与智能整理系统 · 生产级部署清单

> 本文档面向甲方运维 / 实施方，给出系统**生产环境**所需的服务器、软件、网络、安全的**完整配置清单**。
>
> 系统架构：单台主服务器即可承载（Go 主服务 + Python 微服务 + Docker 容器）；如规模扩大可水平拆分。

---

## 一、推荐硬件配置

### 方案 A · 单机部署（≤ 50 用户 / ≤ 200 信息源 / 日均 ≤ 5000 篇）

| 资源 | 推荐配置 | 最低配置 |
|---|---|---|
| **CPU** | 8 核 (Intel Xeon Silver / AMD EPYC 同级) | 4 核 |
| **内存** | 16 GB | 8 GB |
| **系统盘** | 200 GB SSD | 100 GB SSD |
| **数据盘** | 500 GB SSD（用于文章正文 + RSS 缓存 + 日志） | 200 GB |
| **网络** | 100 Mbps 公网出口，1 Gbps 内网 | 10 Mbps |

### 方案 B · 中等规模（50-200 用户 / 1000 信息源 / 日均 ≤ 50000 篇）

| 角色 | 资源 |
|---|---|
| 主应用服务器 | 16 核 / 32 GB / 200 GB SSD |
| 数据库 + 缓存服务器 | 8 核 / 32 GB / 500 GB SSD |
| 采集微服务专用 | 8 核 / 16 GB / 100 GB SSD（Chromium 浏览器实例） |

### 方案 C · 大规模（≥ 500 用户 / ≥ 5000 信息源）

需要 K8s 集群 + 独立数据库（PostgreSQL/MySQL）+ Redis 缓存层 + 对象存储（MinIO/S3）；详情另议。

## 四、端口规划

| 端口 | 服务 | 暴露范围 | 协议 |
|---|---|---|---|
| **443** | Nginx HTTPS | 公网 / 内网（按需） | TCP |
| **80** | Nginx HTTP → HTTPS 301 | 公网 | TCP |
| 8080 | 党政办 Go 主服务 | **仅本地 127.0.0.1** | TCP |
| 8070 | crawl4ai 采集微服务 | **仅本地 127.0.0.1** | TCP |
| 8090 | wechat2rss 容器 | **仅本地 127.0.0.1** | TCP |
| 22 | SSH | 仅运维 IP（白名单） | TCP |

**防火墙规则**：除 443/80/22 外，所有应用端口**禁止对外暴露**，只接受本地回环 + 内网访问。

## 六、对外服务（Nginx 反代）

### 配置文件 `/etc/nginx/sites-available/dzb`

```nginx
# HTTP → HTTPS 强制跳转
server {
    listen 80;
    server_name dzb.example.gov.cn;
    return 301 https://$host$request_uri;
}

# HTTPS 主入口
server {
    listen 443 ssl http2;
    server_name dzb.example.gov.cn;

    ssl_certificate     /etc/ssl/dzb/fullchain.pem;
    ssl_certificate_key /etc/ssl/dzb/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_ciphers         ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;

    # 安全头
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' https://unpkg.com 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; connect-src 'self';" always;

    # 上传 / 文章正文大小限制
    client_max_body_size 10M;

    # 主应用
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 30s;
        proxy_read_timeout 120s;
    }

    # 登录接口加速率限制（同 IP 1 分钟最多 20 次）
    location = /api/auth/login {
        limit_req zone=login burst=5 nodelay;
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 静态资源缓存
    location ~ ^/static/.*\.(js|css|png|jpg|svg)$ {
        proxy_pass http://127.0.0.1:8080;
        proxy_cache_valid 200 1h;
        expires 1h;
        add_header Cache-Control "public, immutable";
    }

    # 健康检查
    location = /health {
        proxy_pass http://127.0.0.1:8080/api/auth/me;
        access_log off;
    }
}

# 限流配置（写到 /etc/nginx/nginx.conf 的 http 块里）
# limit_req_zone $binary_remote_addr zone=login:10m rate=20r/m;
```

## 七、systemd 服务定义

### 1. 党政办主服务 `/etc/systemd/system/dzb-app.service`

```ini
[Unit]
Description=DZB Info Tracking System
After=network.target docker.service dzb-crawler.service
Wants=dzb-crawler.service

[Service]
Type=simple
User=dzb
Group=dzb
WorkingDirectory=/opt/dzb
ExecStart=/opt/dzb/bin/dzbdemo
Restart=on-failure
RestartSec=5s

# 环境变量（生产值从安全位置加载）
EnvironmentFile=/etc/dzb/app.env

# 日志
StandardOutput=append:/opt/dzb/logs/app.log
StandardError=append:/opt/dzb/logs/app.log

# 安全加固
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ReadWritePaths=/opt/dzb/data /opt/dzb/logs
ProtectHome=true

[Install]
WantedBy=multi-user.target
```

### 2. 采集微服务 `/etc/systemd/system/dzb-crawler.service`

```ini
[Unit]
Description=DZB Crawl4AI Microservice
After=network.target

[Service]
Type=simple
User=dzb
Group=dzb
WorkingDirectory=/opt/dzb/deploy/crawler
ExecStart=/opt/dzb/deploy/crawler/venv/bin/python service.py
Restart=on-failure
RestartSec=10s

Environment="CRAWLER_HOST=127.0.0.1"
Environment="CRAWLER_PORT=8070"

StandardOutput=append:/opt/dzb/logs/crawler.log
StandardError=append:/opt/dzb/logs/crawler.log

NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

### 3. wechat2rss 容器走 docker-compose 自启即可（已配置 `restart: unless-stopped`）

## 八、备份策略

### 必备份内容

| 数据 | 频率 | 保留 |
|---|---|---|
| `/opt/dzb/data/data.json` | **每小时** 1 次（rsync 增量） | 30 天 |
| `/opt/dzb/deploy/wechat2rss/data/` | 每日 1 次 | 30 天 |
| `/etc/dzb/app.env` | 每次修改后 | 永久（加密） |
| Nginx 配置 | 每次修改后 | 永久 |
| `/opt/dzb/logs/` | 每日归档（logrotate） | 90 天 |

### 备份脚本示例 `/opt/dzb/backup/backup.sh`

```bash
#!/usr/bin/env bash
set -e
DT=$(date +%Y%m%d-%H%M)
DEST=/opt/dzb/backup/$DT
mkdir -p $DEST
cp /opt/dzb/data/data.json $DEST/data.json
tar czf $DEST/wechat2rss-data.tgz -C /opt/dzb/deploy/wechat2rss data
# 异地备份
rsync -az $DEST backup-server:/backup/dzb/
# 保留 30 天
find /opt/dzb/backup -maxdepth 1 -type d -mtime +30 -exec rm -rf {} \;
```

cron：

```cron
0 * * * * /opt/dzb/backup/backup.sh >> /opt/dzb/logs/backup.log 2>&1
```

## 九、日志与监控

### logrotate 配置 `/etc/logrotate.d/dzb`

```
/opt/dzb/logs/*.log {
    daily
    rotate 90
    compress
    delaycompress
    missingok
    notifempty
    create 0640 dzb dzb
    postrotate
        systemctl reload dzb-app >/dev/null 2>&1 || true
    endscript
}
```

## 十一、首次部署流程（清单）

```bash
# 1. 创建系统用户
sudo useradd -r -s /bin/bash -m -d /opt/dzb dzb

# 2. 安装依赖
sudo apt update
sudo apt install -y nginx docker.io docker-compose-v2 python3.12 python3.12-venv \
                    chromium-browser logrotate certbot

# 3. 下载 Go
curl -fSL https://go.dev/dl/go1.23.4.linux-amd64.tar.gz -o /tmp/go.tgz
sudo tar -C /usr/local -xzf /tmp/go.tgz
echo 'export PATH=/usr/local/go/bin:$PATH' | sudo tee /etc/profile.d/go.sh

# 4. 拉取代码
sudo -u dzb git clone <repo> /opt/dzb/src
cd /opt/dzb/src

# 5. 编译主服务
sudo -u dzb /usr/local/go/bin/go build -o /opt/dzb/bin/dzbdemo .

# 6. 部署采集微服务
sudo -u dzb python3.12 -m venv /opt/dzb/deploy/crawler/venv
sudo -u dzb /opt/dzb/deploy/crawler/venv/bin/pip install crawl4ai fastapi uvicorn pydantic

# 7. 部署 wechat2rss（按 deploy/wechat2rss/README.md）

# 8. 写环境变量
sudo mkdir -p /etc/dzb
sudo cp /opt/dzb/src/deploy/app.env.example /etc/dzb/app.env
sudo chmod 600 /etc/dzb/app.env
sudo vim /etc/dzb/app.env   # 填生产值

# 9. 配置 systemd
sudo cp /opt/dzb/src/deploy/systemd/*.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now dzb-crawler dzb-app

# 10. 配置 Nginx + SSL
sudo cp /opt/dzb/src/deploy/nginx/dzb.conf /etc/nginx/sites-available/dzb
sudo ln -s /etc/nginx/sites-available/dzb /etc/nginx/sites-enabled/
sudo certbot --nginx -d dzb.example.gov.cn  # 或导入政府证书
sudo nginx -t && sudo systemctl reload nginx

# 11. 首次登录 admin/dzb2025 → 立刻改密码

# 12. 配置 cron 备份
sudo crontab -u dzb -e
# 添加: 0 * * * * /opt/dzb/backup/backup.sh

# 13. 健康检查
curl -k https://dzb.example.gov.cn/health
```

## 十四、应急 SOP

| 场景 | 处置 |
|---|---|
| Go 服务挂 | `systemctl restart dzb-app` |
| 采集微服务挂 | `systemctl restart dzb-crawler` |
| wechat2rss 容器挂 | `cd deploy/wechat2rss && ./deploy.sh restart` |
| 磁盘满 | 清理 `/opt/dzb/backup/` 老备份 + `/opt/dzb/logs/` 老日志 |
| 全部丢数据 | `cp /opt/dzb/backup/<最近备份>/data.json /opt/dzb/data/ && systemctl restart dzb-app` |
| DeepSeek 不可用 | 系统自动降级到本地规则引擎，业务不中断 |
| 公众号被风控 | wechat2rss 控制台用新小号重新扫码登录 |
